# AI 健身教练 Agent · 数据驱动的减脂/增肌框架

一个基于 **IDE Chat（Cursor / Windsurf 等）+ 本地 Markdown 仓库** 的通用健身教练项目：

- **你提供**：个人信息、器械清单、饮食/训练偏好、每周实际执行情况。
- **AI 教练提供**：结合你短期与长期目标的周期化训练+饮食规划、每周周课表，以及伴随式的知识讲解和根据执行结果的细腻调整。
- **使用方式**：不需要自己配置 API Key，只要在支持模型的 IDE（如 Cursor / Windsurf）里打开本仓库，用聊天框和「教练」对话即可。

> 本仓库最初来源于一个真实减脂案例：从 98.1kg → 88kg（持续减脂中），同时明显提升力量与围度。通过教练 Agent 做精细化饮食规划，进入了减脂+增肌并行的「romantic 体态重塑区间」。原始个人记录已抽象为「示例案例」，你可以完全替换为自己的数据与目标。

---

## 🧠 项目定位 & 适用人群

- **项目定位**：
  - 这是一个「可以被 AI 教练读取的健身计划仓库模板」，而不是某一个人的周记。
  - 通过统一的目录结构 + 规则文件，约束 AI 输出「可执行、可追踪、可复盘」的训练与饮食计划。

- **适用人群**：
  - 已有健身房/家用器械，希望 **系统化减脂/增肌**。
  - 喜欢用 Markdown 或 Git 管理生活/训练记录。
  - 使用 **Cursor / Windsurf / 其他带 Chat 的 IDE**，希望 **免配置 API Key，直接在本地用大模型当教练**。

## 🚀 如何使用（概要）

> 详细的一次性配置与推荐工作流，请参见：[`docs/get_start.md`](docs/get_start.md)。

在完成 `docs/get_start.md` 中的基本配置后，你就可以在 IDE 里像和真人教练聊天一样使用本项目。

### 在 IDE 聊天框召唤「教练」

在 Cursor / Windsurf 的 Chat 中（确保当前工作区是本仓库）：

- 不需要任何固定口令，你可以像和真人教练聊天一样直接开口，例如：
  - `今天上班好累，只想练 20 分钟，帮我把今天的训练简单改一下。`
  - `这两天聚餐多，体重有点回弹，你帮我看看要不要调一调这周的饮食。`
  - `最近下背有点不舒服，你帮我看看这两周的训练安排要不要改一改。`
  - `下周要出差一周，只能在酒店房间练，你帮我排一版出差期间的训练和大致饮食。`
  - `最近体重掉得慢，但是力量在涨，你帮我看看这样算不算正常，需要调整吗？`
  
简单说，你能想到让真人教练帮你规划的事情，基本都可以直接丢给这个 Agent 教练。

**无需额外配置 API Key**：只要 IDE 能正常使用模型，本项目就能工作。
