# 项目规则

## 目录结构与职责
- `README.md`：项目总览（基本数据、阶段目标、全局策略、监测指标、里程碑）。**仅放原则，不放具体周课表**。
- `计划/运动/`：按周管理，文件名 `第XX周-运动.md`，内含本周具体运动安排（周二至周日训练，周一休息）。
- `计划/饮食/`：按周管理，文件名 `第XX周-饮食.md`，内含本周饮食方案。

## 命名规范
- 周序号：两位数字（`第01周`、`第02周`…）
- 日期范围：文档首行标注（周二至下周一，如 `2025-10-12 ~ 2025-10-18`）
- 语言：中文简体
- 频率：练六休一（周二至周日训练、周一休息）
- **训练周定义**：训练周同样为7天周期，但不从自然周周一开始。当前设定为自然周的周二作为训练周第一天，至下周一为训练周结束（即：周二、周三、周四、周五、周六、周日为训练日，周一为休息日）

## 内容规范

### 运动计划格式
- **设备约束**：所有训练动作必须基于 `运动/设备/` 文件夹中实际可用的器械来制定，严禁随意指定不存在的设备
- 训练日：包含热身、主训练（组×次×负重）、间歇、拉伸、总时长；强度标注 RPE 或心率区间
- 有氧日：区分稳态/间歇，标注时长与目标心率或 RPE
- 课表格式：Markdown 表格，列：`星期`、`主题`、`核心内容`、`时长`、`强度/心率`；内容凝练（如"胸推4×12(50)"）
- **所有无氧动作必须标注目标肌肉**，格式：`动作 组×次(重量)【目标肌肉】`
  - 示例：`胸推4×12(50)【胸+三头】`（主要胸、次要三头）
  - 示例：`下拉4×12(45)【背+二头】`（主要背、次要二头）
  - 示例：`腿伸3×12(30)【股四头肌】`（单一肌群）
  - 热身动作：`划船热身5【背部启动】`

### 饮食计划格式
- **倍数约束**：所有食物克重必须符合实际包装单位的整数倍（如酸奶100g/袋，只能标注100、200、300等；鸡蛋50g/个，只能标注50、100、150等）
- 课表格式：Markdown 表格，列：`星期`、`早餐`、`午餐`、`晚餐`、`加餐`、`备注`；内容凝练（如"鸡胸150+糙米100"）
- 文档顶部标注本周宏量目标（蛋白/碳水/脂肪/总热量）
- 表格下方可补充食材清单、替换方案
- 在 `饮食/食物.md` 中记录常用食物的标准包装规格与营养信息，制定计划时参考

### 通用规范
- 记录区：每周文档末尾留"执行记录"，仅记录真实的最终执行结果与体感，不保留中间修改过程或建议历史
- 调整原则：引用 `README.md` 监测逻辑，不在周文档重复

## 版本与更新
- 当周执行后有重大改动，在文档底部追加"变更记录（日期+要点）"
- 更新 README 全局原则时，不同步修改历史周计划，保持可追溯
- **周课表只在对应周文档中存在，严禁在 README 中维护**

## 工作流
1. `README.md` 维护总策略与原则（不含具体动作安排）
2. 每周在 `计划/运动/第XX周-运动.md` 和 `计划/饮食/第XX周-饮食.md` 制定具体方案（含详细课表）
3. 执行中仅在当周文档记录；双周复盘后更新 `README.md` 全局策略（但不加入课表）
